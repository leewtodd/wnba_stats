# Risk Register: WNBA Analytics Platform

| ID | Risk | Likelihood | Impact | Mitigation | Status | Source |
|---|---|---|---|---|---|---|
| R1 | stats.wnba.com changes API response format for 2026 season | Medium | High | Defensive parsing: log warnings on unexpected fields, don't crash on missing optional fields. wehoop R package community serves as early warning system. Scraper tests use captured JSON fixtures. | Open | Architecture Review |
| R2 | stats.wnba.com rate limits or blocks scraper requests | Low | High | 1.5s delay between calls, proper User-Agent and Referer headers, stagger historical pulls across days. If blocked: adjust delay, rotate User-Agent, or fall back to balldontlie.io API as alternative source. | Mitigated | Architecture Review |
| R3 | Game IDs inconsistent across endpoints (leaguegamelog vs boxscoretraditionalv2 vs boxscoresummaryv2) | Low | High | Validate with sample pulls before full historical ingestion. If inconsistent: build a mapping table using date + team combination as composite key. | Open | Requirements (Open Question #4) |
| R4 | Referee IDs not consistent across seasons in boxscoresummaryv2 | Low | Medium | Validate with sample pulls across 2022-2025. If inconsistent: match referees by name string instead of ID (with fuzzy matching for name variations). | Open | Requirements (Open Question #1) |
| R5 | Claude API generates incorrect SQL that returns plausible but wrong results | Medium | Medium | Query logging enables review. Include example question→SQL mappings in the system prompt to improve accuracy. Over time, review query_log to identify common failure patterns and update the system prompt. | Accepted | Architecture Review |
| R6 | Missed daily scraper runs accumulate (laptop closed for extended period) | Medium | Low | Auto-recovery pulls all games since last successful date. Even a 30-day gap recovers in one run. No data loss possible — the API serves historical data indefinitely. | Mitigated | Architecture Review |
| R7 | Arena relocations not captured in seed data | Low | Low | Manual update process. WNBA arena changes are publicly announced well in advance. Check annually before season start. | Accepted | Domain Model Review |
| R8 | Streamlit performance with large DataFrames or complex Plotly charts | Low | Medium | At 4 seasons of data (~27K player_game_stats rows), DataFrames are small. Plotly handles this easily. If performance degrades: add pagination, limit date ranges in queries, or pre-aggregate in Postgres views. | Accepted | Architecture Review |
| R9 | Claude API pricing changes make NLP interface cost-prohibitive | Low | Low | Current pricing makes even heavy use negligible. If pricing changes: the structured UI (Streamlit dropdowns + pre-built charts) covers all must-have functionality without the API. NLP is additive, not essential. | Accepted | Architecture Review |
